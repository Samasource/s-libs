expectRequest

  • expectRequest<ResponseBody>(method: HttpMethod, url: string, options?: RequestOptions & { body?: null | string | number | boolean | Object | ArrayBuffer | Blob | (null | string | number | boolean | Object)[] }): SlTestRequest<ResponseBody>
    • Defined in projects/ng-dev/src/lib/test-requests/expect-request.ts:51

    This convenience method is similar to HttpTestingController.expectOne(), with extra features. The returned request object will automatically trigger change detection when you flush a response, just like in production.

    This method is opinionated in that you must specify all aspects of the request to match. E.g. if the request specifies headers, you must also specify them in the arguments to this method.

    This method only works when you are using an AngularContext.

    const ctx = new AngularContext();
    ctx.run(() => {
      ctx
        .inject(HttpClient)
        .get('http://example.com', { params: { key: 'value' } })
        .subscribe();
      const request = expectRequest<string>('GET', 'http://example.com', {
        params: { key: 'value' },
      });
      request.flush('my response body');
    });

    spy(1, 2);
